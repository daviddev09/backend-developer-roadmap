# День 116: Базовый CRUD — Добавление и Чтение данных (Create & Read)
## Теория:
* Паттерн Unit of Work: как объект AsyncSession накапливает изменения и отправляет их в базу данных единой транзакцией.
* Добавление данных (Create): использование методов session.add(obj) для одной записи и session.add_all([obj1, obj2]) для списка объектов.
* Чтение данных (Read): современный синтаксис генерации запросов через select(Model).
* Выполнение запросов: метод session.scalars(query).all() для получения списка объектов и session.scalar_one_or_none(query) для безопасного получения одной записи (или None, если запись не найдена).
* Разница между методами завершения транзакции: commit() (зафиксировать в БД), flush() (отправить SQL в БД, но не фиксировать транзакцию, полезно для получения сгенерированного ID до коммита) и refresh() (обновить атрибуты объекта в памяти актуальными данными из БД).
## Практика:
* Напиши скрипт для инициализации AsyncSession.
* Создай три новых объекта модели User и добавь их в сессию через session.add_all().
* Вызови await session.commit(). Сразу после коммита объект в памяти "протухает" (асинхронная специфика). Сделай await session.refresh(user) для первого 
пользователя и выведи его сгенерированный id.
* Напиши запрос select(User), выполни его через session.scalars(), пройдись циклом по результату и выведи имена всех пользователей в консоль.

## Конспект изученных материалов
### 1. Паттерн Unit of Work
* **Своими словами:** Паттерн Unit of Work это что-то типа транзакции на уровне ORM. Его идея такая: Не делаем SQL запросов пока не выполняться все задачи (Добавление пользователя, добавление задач, ...) это значит что пока не будет await session.commit() SQL запрос не сгенерируется и в БД ничего не сохраниться
### 2. Добавление данных (Create)
* **Своими словами:** Когда мы создаём пользователя в Python и передаём его в session.add() новички могут подумать что сейчас это сгенерирует SQL запрос к БД, но это не так. Когда мы передаём объект пользователя, оно сохраняет его в сессию. session.add() используется для создания только одного объекта, а .add_all([]) передаётся список объектов и он сохраняет их всех в сессию
### 3. Чтение данных (Read)
* Запросы на чтение делаются с помощью функии query = select() (В старой версии session.query()) в него передаётся модель (User) а потом эта функция передаётся в функцию (session.scalars(query)).all() эта функция извлекает первый элемент из каждого кортежа и возвращает список объектов Python.
### 4. Разница методов завершения транзакции
* **1. flush():** используется до коммита, но после .add(), генерирует SQL запрос к базе, база принимает данные, выдаёт пользователю id но она остаётся в процессе транзакции, пока нету коммита данные не сохраняются навсегда. Очень хорошая функция когда нужно получить id пользователя чтобы использовать для задачи и т.п.
* **2. commit():** Берёт из сессии все данные и генерирует SQL запросы к базе и в случае успешного выполнения всех SQL запросов делает коммит и завершает транзакцию, а данные сохраняются навсегда.
* **3. refresh():** Используется после коммита для безопасного получения новых данных пользователя после сохранения или обновления в БД.
