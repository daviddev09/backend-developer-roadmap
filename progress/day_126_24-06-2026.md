# День 126: Интеграция Access JWT в FastAPI (Защита эндпоинтов)
## Теория: Механизмы авторизации в HTTP
* Заголовок Authorization: Стандарт передачи токенов в HTTP-запросах (формат Bearer <token>).
* Dependency Injection в FastAPI для Auth: Как работает встроенный класс OAuth2PasswordBearer и почему он делает интеграцию со Swagger UI (эндпоинт /docs) магически удобной.
* Обработка исключений JWT: Изучение ошибок ExpiredSignatureError (когда время exp вышло) и InvalidTokenError (когда подпись не совпадает или токен поврежден).
## Практика: Построение замкнутого цикла авторизации
* Создайте эндпоинт /login (строго по стандарту OAuth2 он должен принимать username и password через Form-data). Эндпоинт должен проверить пароль через verify_password и, если всё верно, вернуть JSON вида: {"access_token": <ваши_токен>, "token_type": "bearer"}.
* Напишите зависимость (Dependency) get_current_user. Она должна:
* Извлекать токен из заголовка.
* Расшифровывать его (через jwt.decode).
* Извлекать user_id из полезной нагрузки.
* Делать запрос в базу данных SQLAlchemy, чтобы найти пользователя.
* Возвращать объект пользователя (или выбрасывать HTTP 401 Unauthorized, если что-то пошло не так).
* Создайте защищенный эндпоинт /users/me, который принимает зависимость get_current_user и возвращает информацию о текущем авторизованном пользователе. Проверьте работу через Swagger (появится кнопка "Authorize" с замочком).

## Конспект изученных материалов
### 1. Заголовок Authorization
* Когда клиент отправляет запрос к защищённому эндпоинту, он должен передать полученный JWT токен
* Передавать токен в Query параметрах плохая практика, так как адреса запросов логируются серверами и провайдерами, что может привести к утечке токена
* Стандарт индустрии - передача токена в HTTP-заголовке Authorizatiion с типом схемы Bearer (В переводе - предъявитель, или носитель) **Формат загаловка** выглядит так: Authorization: Bearer <твой_jwt_токен>
* Слово **Bearer** сообщает серверу: Всё, что идёт следом через пробкл это JWT-токен доступа. Проверь его подпись
### 2. Dependency Injection OAuth2PasswordBearer
* Это инструмент предоствляемый FastAPI класс OAuth2PasswordBearer из модуля fastapi.security
#### Как работает
* **1.** Мы инициализируем объект класса, передавая ему эндпоинт, на котором выдаётся токен: oauth2scheme = OAuth2PasswordBearer(tokenUrl='/login')
* **2.** Мы используем этот объект в качестве заисимости во всех защищенных эндпоинтах: token: str = Depends(oauth2_scheme)
* **3.** При запросе FastAPI автоматически ищет заголовок Authorization, проверяет, начинается ли он со слова Bearer. Извлекает токен и передаёт его аргументом в функцию. Если заголовок отсутствует или пустой он не пустит запрос дальще и выбросит ошибку: 401 Unauthrized
* Как только ты добавишь Depends(oauth2_scheme) хотя бы в один эндпоинт, в правом верхнем углу интерфейса Swagger UI появится заветная кнопка «Authorize» с замочком. Ты сможешь один раз ввести туда свой токен, и Swagger будет автоматически подставлять заголовок Authorization: Bearer <токен> во все твои тестовые запросы. Это невероятно упрощает ручное тестирование API!
### 3. Обработка исключений JWT
* Токен, извлеченный из заголовка это просто строка. Нужно проверить её подпись и прочитать payload. При декодировании могут возникнуть три сценария
* **1. успех:** Подпись верна, время жизни не вышло, извлекается user_id из поля sub
* **2. jwt.ExpiredSignaturedError:** Подпись верна но время жизни истекло
* **3. jwt.InvalidTokenError:** Подпись повреждена, токен подделан хакером или передан не JWT
