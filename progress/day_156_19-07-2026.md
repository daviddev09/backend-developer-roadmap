# День 156 (19.07.2026): Интерактивный Rebase (Interactive Rebase)
## Теория:
* Как объединить несколько мелких коммитов в один (Squash / Fixup).
* Как изменить текст старого сообщения коммита (Reword).
* Как удалить неудачный коммит из истории (Drop).
## Практика:
* В своей рабочей ветке сделай 4 мелких бессмысленных коммита (например, "wip", "fixed typo", "add part 1", "add part 2").
* Запусти git rebase -i HEAD~4.
* В открывшемся редакторе поменяй команду с pick на squash (или s) для последних трех коммитов, чтобы "схлопнуть" их в первый.
* Сохрани, напиши одно красивое итоговое сообщение и проверь лог.

## Конспект изученных материалов
### Интерактивный rebase
* Интерактивный rebase это мощный инструмент Git, который позволяет полностью переписать, отредактировать, объединять или удалить коммиты в истории ветки перед тем как показывать кому-то или объединять в main. 
* Если обычный rebase просто переносит коммиты "как есть" на новое основание, то интерактивный режим открывает текстовый редактор со списком коммитов и позволяет выдать команду для каждого из них.
* Интерактивный rebase позволяет:
    * менять историю
    * удалять коммиты
    * объединять коммиты
    * менять порядок
    * изменить сообщения
    * разделять коммиты
* **Основная команда:** *git rebase -i HEAD~3*
    * Что такое HEAD: *HEAD* это указатель на текущий коммит, например: A -> B -> C -> D. Если написать: *HEAD~1* получиться *С* родитель HEAD, Если 2, то B
    * Что такое **-i**: флаг *-i* означает "interactive"
### 1. Объединение мелких коммитов (Squash / Fixup)
* В процессе работы нормально делать много мелких коммитов, но, перед отправкой на ревью эти куски лучше собрать в один красивый коммит "Создание формы обратной связи". Для этого нужны squash и fixup:
    * **Squash(s)** Берет текущий коммит и "вливает" его в коммит выше (то есть в предыдущий). Git откроет новое окно редактора, чтобы ты мог объединить сообщения обоих коммитов в одно новое.
    * **Fixup(f)** Делает то же самое, что и squash, но игнорирует сообщение вливаемого коммита. Это идеально для коммитов типа "исправил опечатку", где текст коммита абсолютно не несет смысловой нагрузки.
### 2. Изменение текста старого коммита (Reword)
* Чтобы поменять сам текст, не трога сам код, используется **reword**:
    * **Reword(r):** Оставляет код коммита без изменений, но останавливает процесс перебазирования и открывает редактор, чтобы вписать туда текст
### 3. Удаление неудачного коммита (Drop)
* Используется чтобы удалить коммит который не нужен:
    * **Drop(d):** Полностью удаляет коммит из истории. Альтернативный способ - просто удалить саму строку с этим коммитом из текстового редактора. Эффект будет таким же
> #### Для отмены использовать: *git rebase --abort*
