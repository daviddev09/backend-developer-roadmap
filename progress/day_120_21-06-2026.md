# День 120: Интеграция CRUD и Пагинации с FastAPI
## Теория:
* Разделение слоев: почему категорически нельзя возвращать объекты моделей SQLAlchemy напрямую из эндпоинтов FastAPI клиенту.
* Паттерн DTO (Data Transfer Object) через Pydantic. Создание схем для валидации входящих данных (например, TaskCreate) и форматирования исходящих (TaskRead).
* Настройка Pydantic-схем: использование model_config = ConfigDict(from_attributes=True). Это позволяет Pydantic автоматически извлекать данные из атрибутов ORM-объектов.
* Передача параметров фильтрации и пагинации через стандартные Query-параметры FastAPI (limit: int = 10, offset: int = 0).
* Инъекция зависимостей (Dependency Injection) FastAPI: передача асинхронной сессии базы данных (AsyncSession) внутрь эндпоинта через Depends().
## Практика:
* Создай Pydantic-схемы: TaskCreate (только title и description) и TaskRead (включает id, status и настроенный from_attributes=True).
* Напиши POST-эндпоинт /tasks/, который принимает схему TaskCreate, преобразует её в ORM-модель Task, сохраняет через внедренную сессию в базу и возвращает Pydantic-схему TaskRead с готовым id.
* Напиши GET-эндпоинт /tasks/, который принимает Query-параметры status (строка), limit (число) и offset (число). Эндпоинт должен фильтровать задачи по статусу (если он передан), применять пагинацию и возвращать список схем list[TaskRead].

## Конспект изученных материалов
### 1. Разделение слоев: Почему нельзя возвращать объекты ORM
* **Утечка данных:** Если возвращать клиенту данные напрямую из ORM объекта, то можно нечаянно вернуть хеш пароля
* **Ошибка сериализации JSON:** FastAPI под капотом попытается превратить ответ в JSON, но объекты ORM это сложные сущности со своими скрытыми атрибутами и поэтому будет ошибка
* **Бесконечная рекурсия:** Если есть двунаправленная связь сериализатор попадет в бесконечный цикл и сервер упадёт.
### 2. Паттерн DTO
* **Идея:** База данных взаимодейтвует только с внутренним слоём приложения, а на выход, к клиенту мы отдаём специально подготовленные данные. И это Pydantic схемы
### 3. Настройка Pydantic схем
* Для того чтобы настроить Pydantic схему чтобы он мог брать объекты ORM, нам в схеме (TaskRead) нужно объявить метод model_config = ConfigDict(from_attributes=True). Эта строчка говорит Pydantic "Если тебе передали не словарь, а объект, не паникуй, попробуй просто достать данные через атрибуты"
