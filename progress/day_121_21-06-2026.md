# День 121: Финализация бизнес-логики и Глобальная обработка ошибок
## Теория:
* Паттерн обработки исключений в FastAPI: как работают глобальные обработчики (Exception Handlers) и зачем они нужны.
* Класс HTTPException: правильная генерация кастомных ошибок с детализацией (detail) и нужными статус-кодами.
* Настройка CORS (Cross-Origin Resource Sharing): почему браузеры блокируют запросы с фронтенда к твоему API и как правильно настроить CORSMiddleware в FastAPI для работы с React/Vue.
* Рефакторинг: проверка структуры проекта, убеждаемся, что Router, Service и Repository строго разделены.
## Практика:
* Допиши все недостающие эндпоинты твоего Blog API (создание постов, комментариев, лайков — в зависимости от твоей схемы).
* Создай глобальный обработчик ошибок (например, для перехвата исключений, когда запись не найдена в базе данных, чтобы отдавать красивый JSON с 404 Not Found вместо падения сервера).
* Добавь CORSMiddleware в главный файл main.py и разреши запросы с http//localhost:3000 (стандартный порт для локального фронтенда).
---
## Конспект изученных материалов
### 1. Глобальные обработчики исключений
* **Своими словами:** пишутся как обычный класс, наследуются от базового класса Exception один раз объявляются в проекте, и когда есть ошибка вызывается raise EntityNotFound(detail='...'). Глобальные обработчики исключений умеют обрабатывать ошибки в любом месте приложения, и когда в проекте много эндпоинтов они очень удобны
### 2. Класс HTTPException
* **Своими словами:** Сам FastAPI предоставляет нам класс HTTPException, использовать их не выгодно когда в проекте очень много эндпоинтов, и если каждый раз писать длинные raise HTTPException(status_code=....) проект превратиться в кашу из проверок и поэтому надо использовать глобальные обработчики исключений
### 3. Настройка CORS 
* Когда фронтенл разработчик напишет интерфейс на React или Vue, Фронтенл будет запущен на сервере http//localhost:3000, а бэкенд запущен на http//localhost:8000
* Если React попытается отправить POST /posts/ FastAPI, браузер заблокирует этот запрос и выдаст страшную красную ошибку CORS в консоли.
* **Почему?** Это механизм безопасности браузеров. Браузер запрещает сайтам с одного домена обращаться к API другого домена, если бэкенд явно не дал на это разрешение.
* **Решение:** Добавить CORSMiddleware. Middleware — это программа-посредник, через которую проходят абсолютно все входящие и исходящие запросы сервера. Мы должны явно сказать ей: "Пропускай запросы с порта 3000".
### 4. Структура проекта
* **Repository:** умеет общаться с Базой данных, он полностю отделён от бизнес логики приложения
* **Service:** принимает данные от роутера, делает проверки на логику, и может вызвать репозиторий и передать данные или попросить
* **Router:** приниает HTTP запрос а также возвращает ответ
