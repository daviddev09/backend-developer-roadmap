# День 118 (21.06.2026): Фильтрация данных (WHERE на уровне ORM)
## Теория:
* Метод .where() как полный аналог оператора WHERE в чистом SQL. Его привязка к конструкции select().
* Использование базовых операторов сравнения Python (==, !=, <, >) прямо с атрибутами моделей (например, Task.status == "done").
* Специфичные ORM-операторы для поиска по строкам: методы .like(), .ilike() (поиск без учета регистра) и их отличие.
* Оператор .in_() для проверки вхождения значения в список (аналог SQL IN).
* Построение сложных логических фильтров: импорт и использование функций and_(), or_(), not_() из модуля sqlalchemy для комбинирования нескольких условий.
## Практика:
* Напиши запрос, который вытащит из базы только те задачи, у которых статус равен "done".
* Найди всех пользователей, чьи имена заканчиваются на "ov" (используя User.name.ilike("%ov")).
* Скомбинируй условия через or_(): найди задачи, у которых статус либо "new", либо "in_progress".
* Найди задачи, принадлежащие пользователям с конкретными ID, передав список в метод .in_([1, 2, 3]).

## Конспект изученных материалов
### 1. Метод .where()
* **Синтаксис:** Пишется select(Task).where(Task.id == 2). В обычном Python оператоп ==  вычислял бы True/False, но когда ты пишешь такой запрос SQLAlchemy перехватывает его и вместо вычисление генерирует кусокSQL запроса
### 2. Базовые операторы
* **Равно:** ==
* **Не равно:** !=
* **Меньше:** < **Больше:** >
* **Меньше либо равно:** <=
* **Больше либо равно:** >=
* **Пример:** select(Task).where(user_id == 2)
### 3. Операторы поиска
* Используются для поиска совпадений, и у этих методов есть свои символы-джокеры
* **%** означает: любое количество любых символов
* **_** означает: любой один символ
* **.like('%ov'):** Регистрозависимый поиск (Case sensitive) он найдёт Ivanov, но не найдет IVANOV, потому что заглавные и строчные буквы для него - это разные символы
* **.ilike('%ov%):** РегистроНезависимый поиск (Case insensitive). Буква i в начале означает: insensitive, регистр ей безразличен
### 4. Оператор .in_()
* **Синтаксис:** select(Task).where(Task.user_id.in_([1, 2, 3])). Используется, например если нужно получить задачи пользователей, вместо множества .or_ пишем .in_ SQLAlchemy генерирует SQL запрос превращая её в IN(1, 2, 3)
### 5. Сложные логические фильтры
* **Оператор and_():** По умолчанию если передать в .where() несколько аргументов через запятую то он сам свяжет их с AND
* **Оператор or_():** Она принимает внутрь себя неограниченное количество условий и говорит базе: "Верни строку, если совпадает хотя бы одно из этих условий". **Синтаксис:**stmt = select(Task).where(or_(Task.status == 'new', Task.status == 'in_progress'))
