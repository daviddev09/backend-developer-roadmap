## Конспект изученных материалов
### 1. Стандарт OpenAPI
* **OpenAPI** (ранее известный как Swagger Specification) — это международный стандарт
и спецификация для описания REST API. Он представляет собой структурированный файл в формате JSON или YAML, который описывает:
* Какие эндпоинты (URL) есть у твоего сервера.
* Какие HTTP-методы (GET, POST, PUT, DELETE) они поддерживают.
* Какие параметры (Query, Path, Headers) и тела запросов (Request Body) они принимают.
* Какие статус-коды и структуры данных они возвращают в качестве ответов.
### 2. Интерактивные интерфейсы
* **Swagger UI (/docs):** Это самый популярный инструмент. Он представляет собой интерактивную панель, где каждый эндпоинт можно раскрыть, изучить его структуру и, самое главное, протестировать прямо из браузера с помощью кнопки "Try it out".
* **Redoc (/redoc):** Это альтернативный, более строгий и минималистичный трехпанельный интерфейс. Он отлично подходит для чтения спецификации, поиска и изучения структуры данных, но в отличие от Swagger, в нем нельзя выполнять "живые" запросы к серверу.
### 3. Обогащение документации
* Чтобы документация не выглядела как безликий список URL-адресов, мы должны явно подсказывать FastAPI метаданные. Делается это на двух уровнях: в параметрах декоратора и в самом коде функции.
* **Параметры декоратора роутера:**

* **tags:** Группирует эндпоинты в Swagger. Без тегов все эндпоинты будут свалены в одну кучу default.
* **summary:** Короткое (в одну строчку) описание того, что делает эндпоинт. Если не указать, FastAPI возьмет имя функции и заменит подчеркивания на пробелы.
* **description:** Подробное многострочное описание.
* **response_model:** Указывает Pydantic-схему ответа.
* **status_code:** Дефолтный HTTP-статус при успехе (например, 201 для POST).
* **responses:** Позволяет задокументировать дополнительные ответы (например, ошибки 404, 403, 409) с их структурами.
* Если не хочется писать длинный текст в декораторе, можно написать обычный многострочный комментарий (docstring) внутри функции эндпоинта. FastAPI автоматически заберет его и поместит в поле description в Swagger!
