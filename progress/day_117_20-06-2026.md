# День 117: Базовый CRUD — Обновление и Удаление (Update & Delete)
## Теория:
* ORM-стиль обновления (изменение в памяти): извлечение объекта из базы, изменение его атрибутов напрямую (user.name = "New") и вызов commit(). SQLAlchemy автоматически отслеживает состояние объекта (статус dirty) и сама сгенерирует SQL-команду UPDATE.
* Core-стиль массового обновления: конструкция update(Model).where(...).values(...). Идеально для обновления тысяч строк без загрузки их в оперативную память.
* ORM-стиль удаления: метод session.delete(obj). Требует предварительной загрузки объекта в память. Удобен, когда нужно, чтобы сработали ORM-каскады удаления зависимых сущностей.
* Core-стиль массового удаления: конструкция delete(Model).where(...). Работает напрямую в БД, невероятно быстро, но игнорирует каскады ORM-уровня (зависит только от каскадов самой СУБД).
## Практика:
* Извлеки из базы данных конкретную задачу по её id с помощью scalar_one_or_none().
* Измени статус этой задачи на "done" через обычное присваивание атрибуту и сделай commit().
* Напиши массовый запрос через update(), который поменяет статус всех задач с "new" на "in_progress" за один проход, и выполни его через session.execute().
* Вытащи в память одну задачу и удали её через session.delete(), затем зафиксируй изменения.

## Конспект изученных материалов
### 1. ORM-стиль обновления (ORM-UPDATE)
* **Своими словами:** Это стиль обновления данных, на уровне Python. Для этого мы вытаскиваем данные из базы, меням значение, например tasks.status = 'changed', и SQLAlchemy смотрит и видит что объект изменлся и помечает его как "dirty" и при session.commit делает SQL запрос к базе чтобы изменить значение. Обычно подходит для обновления одного или нескольких объектов, так как при большом количестве данных это будет огромная нагрузка
### 2. CORE-стиль обновления (CORE-UPDATE)
* **Своими словами:** Это стиль обновления данных на уровне Базы Данных. Для этого мы делаем query = update(Task).where(Task.id == n).values(параметр = 'новое значение') и после этого база данных сама выполняет удаление и нагрузки не будет. Обычно используется для массового обновления данных.
### 3.  ORM-стиль удаления (ORM-DELETE)
* **Своими словами:** Надо выгрузить данные из БД и удалить. Используется для удаления одного или нескольких объектов, не подъодит когда много данных, можно отслеживать каскады(relationship(cascades='all, delete-orphan') )
### 4. CORE-стриль удаления (CORE-DELETE)
* **Своими словами:** Очень быстрое удаления, подходит для массового удаления, работает на уровне Postgres, не требует выгрузки объектов из БД. Из минусов нельзя применять каскады ORM, зависит от каскадов самого СУБД (ON DELETE CASCADE)
