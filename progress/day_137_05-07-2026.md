# День 137: Логирование в FastAPI и SQLAlchemy
## Теория:
* Паттерн Middleware в FastAPI: как перехватывать все входящие HTTP-запросы и исходящие ответы для записи их в логи.
* Логирование времени ответа: использование модуля time внутри Middleware для замера скорости работы каждого эндпоинта.
* Управление логами SQLAlchemy: как отключить простой echo=True и перенаправить SQL-запросы в твой собственный, красиво настроенный логгер.
## Практика:
* Создай @app.middleware("http"), который будет логировать: HTTP-метод (GET/POST), путь запроса (например, /users), статус ответа (200/404) и время выполнения в миллисекундах.
* Добавь в настройки SQLAlchemy логирование через стандартный модуль logging (установив уровень логгера sqlalchemy.engine в INFO).
* Сделай запрос к своему API и посмотри в консоль: ты должен увидеть четкую хронологию — пришел HTTP-запрос, выполнился SQL-запрос, запрос завершился за Х миллисекунд.

## Конспект изученных материалов
### 1. Паттерн Middleware
* **Своими словами:** Middleware это промежуточное ПО, через которое проходит HTTP-запрос и ответ. Он проверяет запрос до того как оно попадёт в эндпоинт и может проверять ответ от эндпоинта пока он не отправится. Любой Middleware работает только два раза.
### 2. Логирование времени ответа
* **Своими словами:** Логирование времени ответа нужно для того чтобы находить узкие места в коде. Например, если в какой-то момент программа может ответить клиенту долго, то с помощью логов мы это увидим и можем рефакторить. Для этого в Middleware когда приходит запрос, отмечается время с помощью очень точной функции time.per_counter() и когда уже эндпоинт вохвращает ответ, делаем time.perf_counter - sart_time. И так можно засчитать время обработки запроса сервером
### 3. Управление логами SQLAlchemy
* **В чем проблема echo=True ?** Он просто выводит текст в консоль, этот текст не имеет красивого формата, в нем нет времени, и самое главное этот текст не пишется в файл. В продакшене echo всегда должен быть =False.
* Под капотом **SQLAlchemy** использует модуль logging, в нём зарегистрировано несколько логгеров с именами: **sqlalchemy.engine** - отвечает за логирование SQL-запросов и транзакций, **sqlalchemy.pool** - логирует, как открываются и закрываются подключения к базе, **sqlalchemy.orm** - логирует работу ORM
* **Перенаправить SQL на свой логгер:** Нужно: 1) Отлючить echo, 2) Поймать логгер по его имени, 3) Понизить его уровень на INFO чтобы он начал отдавать SQL-запросы
