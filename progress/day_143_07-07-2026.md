# День 143 (07.07.2026): Слой представления — Routers (Контроллеры)
## Теория:
* "Глупые" роутеры: почему эндпоинт в FastAPI должен занимать не более 5-10 строк кода.
* Роль контроллера: принять HTTP-запрос, получить провалидированную Pydantic-схему, вызвать соответствующий метод Сервиса и вернуть Pydantic DTO клиенту.
* Перехват бизнес-ошибок: как обрабатывать кастомные исключения Сервиса на уровне Роутера (или через глобальные Exception Handlers, как на 121-м дне) и превращать их в понятные HTTP-статусы (400, 404, 409).
* Группировка эндпоинтов: использование APIRouter для выноса логики из main.py в изолированные модули.
## Практика:
* Создай файл api/routers/users.py и инициализируй APIRouter.
* Напиши POST-эндпоинт /users/register, который принимает схему UserCreate и возвращает UserRead.
* Внутри эндпоинта вызови метод register_user из UserService.
* Интегрируй этот роутер в главный файл main.py через app.include_router().

## Конспект изученных материалов
### 1. Глупые роутеры
* **Своими словами:** Иделаьный роутер должен состоять от 3 до 10 строк кода. Если в роутере есть большое количество строк, вычислений, условий то это уже спагетти-код. Роутер должен принимать только HTTP-запрос, вызывать сервис и возвращать ответ клиенту. Если роутер делает всё вместо сервиса то такой роутер становиться сложно тестировать и также это ломает слоистую архитектуру
### 2. Роль контроллера
* **Что такое Controller:** В арихитектурном паттерне MVC (Model-View-Controller) именно Controller принимает входящий запрос и вызывает нужную бизнес-логику. В FastAPI роль Controller выполняет Router
* **Роль контроллера:** Router должен принимать HTTP-запрос, а FastAPI сам провалидиоует JSON на соответствие Pydantic схеме, роутер должен вызвать соответствующий метод сервиса, передать в него данные и вернуть ответ клиенту, а в случае возникновения исключения вернуть понятный HTTPException клиенту.
### 3. Перехват бизнес-ошибок
* **Своими словами:** Допустим сервис обнаружил что адрес почты занят, он выбрасывает кастомное исключение: EmailAlreadyExistsError, а Router перехватывает бизнес ошибку через блок try/except и возвращает клиенту: HTTPException(status_code=..., detail=...) или же перехватывать бизнес-ошибки через Глобальные обработчики
### 4. Группировка эндпоинтов
* **Своими словами:** Когда создаётся объект роутера: *router = ApiRouter(prefix=..., tags=[...])*, здесь *prefix=* это общий путь для роутеров, например, если написать: *prefix='/users'* то во всех эндпоинтах которые мы создадим через этот роутер автоматически будет этот путь. А *tags=[]* это группировка для эндпоинтов чтобы они красиво отображались в SwaggerUI, например, если сделаем: *tags=['Users']* то все эндпоинты этого роутера будут сгруппированы в SwaggerUI под названием Users
