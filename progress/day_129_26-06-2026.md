# День 129: Философия AsyncIO и корутины под капотом
## Теория: 
* Концепция Event Loop (Цикла событий): как одно ядро процессора умудряется управлять тысячами подключений, переключаясь между ними во время пауз (ожидания ответа от сети или БД).

Сравнение подходов к конкурентности в Python:
| Подход | Суть механизма | Идеальный юзкейс |
| :--- | :--- | :--- |
| Синхронность | Задачи выполняются строго по очереди. Код блокируется на каждой строке. | Простые скрипты. |
| Multiprocessing | Запуск независимых процессов операционной системы, обход GIL. | Тяжелая математика, обработка видео (CPU-bound). |
| Multithreading | Потоки внутри процесса. Подвержены блокировке GIL в Python. | Старые I/O приложения. |
| AsyncIO | Один поток жонглирует задачами, пока другие «спят». | Базы данных, API, веб-серверы (I/O-bound). |

* Анатомия асинхронного кода: зачем перед def пишут async (превращение функции в корутину) и почему без вызова await асинхронная магия не работает.
* Кооперативная многозадачность: почему корутина должна добровольно «отдать» управление Циклу событий, чтобы другие функции могли поработать.
## Практика:
* Создай чистый Python-скрипт (без FastAPI).
* Напиши асинхронную функцию download_file(), которая имитирует долгое скачивание через await asyncio.sleep(2).
* Запусти эту функцию 5 раз последовательно и замерь время (должно занять 10 секунд).
* Используй функцию asyncio.gather(...), чтобы запустить эти 5 задач параллельно, и убедись, что общее время выполнения сократилось до 2 секунд.

## Конспект изученных материалов
### 1. Концепция Event loop (Цикла событий)
* **В синхронном** программировании код выполняется строчка за строчкой. Если коду нужно скачать файл из интернета или дождаться ответа от БД, ядро процессора блокируется. Оно ничего не делает, просто ждет ответа от сети
* **Асинхронность** решает эту проблему простоя. В сердце AsyncIO лежит Event Loop (Цикл событий)
* **Event Loop** это бесконечный While True под капотом Python, который постоянно опрашивает оаерационную систему: "Ответ от базы данных пришел? Нет? Окей, выполняю другую задачу. А ответ от API пришел? Да? Отлично, продолжаем выполнять код этой задачи".
### 2. Анатомия асинхронного кода: async def и await
* Асинхронный код в Python строиться на двух ключевых словах
* **1. async def** (Корутина/Coroutine) когда ставиться async перед определением функции, она перестает быть обычной фукцией. Она превращается в корутину.
* **2. await** (Точка паузы) Слово await можно использовать только внутри async def. Оно переводится как "дождаться". Но его истинный смысл для Event Loop звучит так "Эй, Event Loop, здесь мы будем долго ждать ответа от сети. Поставь меня на паузу, иди выполняй другие корутины, а когда ответ придет - разбуди меня, и я продолжу с этой строчки". Без await асинхронная магия не работает. Именно на строчках с await происходит переключение задач
### 3. Кооперативная многозадачность
* **MultiThreading** - это вытесняющая многозадачность. Операционная система сама, жестко и без спроса останавливает один поток и запускает другой
* **AsyncIO** - это кооперативная многозадачность. Event Loop не может прервать функцию силой! Функция должна добровольно сказать: "Я уступаю место другим". Как она это говорит? С помощью слова await
